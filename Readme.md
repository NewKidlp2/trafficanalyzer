Запуск проекта:

	JDK:
1.1) Установите jdk (минимум 8 версия) с сайта: https://www.oracle.com/java/technologies/javase-downloads.html
1.2) Откройте переменные среды: откройте панель управления -> система и безопасность -> система -> дополнительные
параметры системы -> переменные среды.
1.3) Убедитесь, что директория bin JDK добавилась в переменную PATH (системная переменная, для этого необходимо
открыть переменные среды и открыть переменную PATH, если её нет, то необходимо добавить.

	WinPcap:
	
2.1) Загрузите и установите winpcap: https://www.winpcap.org/install/	

	IntellijIdea:
2.1) Загрузите и установите среду разработки с сайта: https://www.jetbrains.com/ru-ru/idea/download/#section=windows

	PostgreSQL:
3.1) Загрузите и установите PostgreSQL с сайта: https://www.postgresql.org/download/
3.2) Запустите PgAdmin4.exe в папке (директория установки postgreSQL\PostgreSQL\10\pgAdmin 4\bin)
(должен быть открыт браузер)
3.3) В брузере откроется интерфейс postgreSQL, установите пароль 1238799.
3.4) Нажмите Object вверху экрана, вкладка create и создайте server c названием
limits, установите владельца как postgres.
3.5) Во вкладке connection установите в строке host name: localhost, порт: 5432,
username: postgres, password: 1238799, во вкладке maintenance database
введите limits. Нажмите Save.

	Заполнение базы данных:
3.1) Войдите в папку проекта и запуститие его, открыв файл build.gradle с помощью IntellijIdea.
Либо откройте командную строку в директории проекта и напишите gradle build.Для этого необходимо
установить gradle.
3.2) После окончания сборки проекта, откройте вкладку Gradle, которая находится справа.
3.3) Нажмите стрелку рядом со вкладкой TrafficAnalyzer, затем tasks, затем flyway и щёлкните дважды
по flyway migrate. После этого в базе создана и заполнена таблица limits_per_hour.
3.3.1) Вместо пункта 3.3 можно запустить в командной строке из директории проекта команду gradle flywayMigrate -i
Для этого также необходим установленый Gradle.
3.4) Для добавления новых данных необходимо во папке проекта по пути
\src\main\resources\db\migration создать скрипт, назвать его по правилу:
Имена должны иметь вид V1_1__some_text.sql для первого скрипта, далее для второго V2__else_text.sql
для второго и так далее. Обязательно обратить внимание на двойное подчеркивание перед текстом. В
нашем случае уже есть два скрипта, поэтому имя третьего будет V3__text.sql. Заполнить его можно по
примеру первых двух файлов.

	Zookeeper:
4.1) Загрузите zookeper (я использовал версию 3.4.14), с сайта: http://zookeeper.apache.org/releases.html
и разархивируйте его в папку zookeeper (т.е будет trafficanalyser\zookeeper) в директории проекта.
4.2) Откройте командную строку, в ней, с помощью команды cd перейдите в директорию
(директория проекта)\zookeeper\zookeeper-3.4.14\bin и затем напишите zkserver.

	Kafka:
5.1) Загрузите kafka с сайта http://kafka.apache.org/downloads и разархивируйте в папку kafka в
директории проекта.
5.2) Как в пункте 4.1, откройте новую командную строку (не закрывая старую!)
в директорию (директория проекта)\kafka\kafka и выполните команду 
.\bin\windows\kafka-server-start.bat .\config\server.properties
5.3) Откройте еще одну командную строку и перейдите как в пункте 4.1 в директорию:
(директория проекта)\kafka\kafka\bin\windows
5.4) Создайте топик с помощью команды:
kafka-topics.bat --create --zookeeper localhost:2181 --replication-factor 1 --partitions 1 --topic alerts
5.5) В этой же командной строке выполните команду:
kafka-console-consumer.bat --bootstrap-server localhost:9092 --from-beginning --topic alerts
Это откроет сообщения, которые идут в топик. Не закрывайте эту командную строку.

	Непосредственно запуск:
6.1)Откройте командную строку в директории проекта и напишите gradle run.
6.2) Для того чтобы передать ip для фильтрации (пункт 4 из задания) введите gradle run --args=ip. К примеру,
gradle run --args=192.0.0.1
